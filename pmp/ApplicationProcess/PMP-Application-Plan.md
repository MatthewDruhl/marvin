## Application Overview

The PMI application has 3 sections:

| Section | What it is | Status |
|---------|-----------|--------|
| Formal Education | Degree(s) attained | ⬜ Gather |
| Professional Education | 35 contact hours certificate | ⬜ Gather |
| Work Experience | Project descriptions (100–500 words each) | ⬜ Draft |

**Experience requirement:**
- Bachelor's degree or higher → **36 months** of PM experience (within last 8 years)
- Associate degree / HS diploma → **60 months** of PM experience (within last 8 years)

**Cost:** PMI member = $524 | Non-member = $555
**Tip:** Joining PMI (~$139/year) saves money if you pay for membership before applying.

## Step 3 — Write Project Descriptions

Each project needs a **100–500 word description** covering exactly 5 sections:

### Required Sections (in order)

1. **Project Objectives** — What was the goal? 1–2 sentences.
2. **Outcome** — What was the result when the project was done? 1–2 sentences.
3. **Role** — "My role was the project manager." Keep it simple.
4. **Responsibilities** — This is the critical part (see below).
5. **Deliverables** — What tangible outputs were produced?

### Writing Responsibilities (the hard part)

**PMI does NOT want to know what physical work you did.**
PMI wants to know what **project management tasks** you performed — in PMBOK or Agile terms.

**Traditional project — think through the 5 process groups:**
- *Initiating:* Created project charter, identified stakeholders
- *Planning:* Collected requirements, defined scope, created WBS, built schedule, set budget, created quality management plan, completed risk assessment (stored in risk register), created resource management plan, analyzed stakeholder communication requirements
- *Executing:* Acquired team, directed and managed project work, managed team conflicts, processed change requests through change management process
- *Monitoring & Controlling:* Reviewed work to prevent scope creep, tracked schedule/budget performance
- *Closing:* Managed sponsor acceptance of deliverables, conducted lessons learned, released team, archived project records

**Agile project — include ceremonies, processes, artifacts from the Agile Practice Guide:**
- Worked with product owner, facilitated daily standups, prioritized product backlog, used burn-up/burn-down charts, Kanban board, served as servant leader

**Hybrid project:** Cover PMBOK terms for traditional phases + Agile ceremonies for iterative phases.

### Tips
- Use **"I"** throughout — PMI only cares what you personally did
- Target **300–400 words** (leaves room to trim or add without going outside 100–500)
- Use PMBOK terminology, not industry-specific jargon
- Grammar check before submitting (show to a colleague if possible)
- Write in Word first, paste into the application

### AI-Assisted Drafting (recommended workflow)

Use the following prompt structure in ChatGPT / Claude:

```
Write me a PMP application project description.

Project Name: [name]
Objective: [what was the goal]
Outcome: [what was the result]
My Role: Project Manager
Responsibilities (what I did in PMBOK terms): [list key PM tasks]
Deliverables: [list tangible outputs]

Instructions: Write a 300–400 word description using only PMBOK terms and processes.
Include the 5 required sections: Project Objectives, Outcome, Role, Responsibilities, Deliverables.
Use first person ("I"). Do not use industry-specific jargon.
```

After generating, review and:
- Remove anything you didn't actually do
- Add anything the AI missed
- Verify it still reads in PMBOK language

## Step 6 — Audit Prep (do this before submitting)

~10% of applicants are audited. You'll know within 5–10 minutes of submitting.
**Prepare these in advance so audit response is fast:**

- [ ] Degree copy saved digitally
- [ ] 35-hour certificate saved digitally
- [ ] For each project: contact person knows they may be asked to sign a verification form
  - Send a quick heads-up email: "I'm applying for my PMP, can I use you as a reference?"
- [ ] If an employer no longer exists: identify a former colleague who can still verify the work

**What happens if audited:**
PMI sends a pre-filled form (with your own info from the application).
You take it to the contact person → they sign it → you send it + degree + certificate to PMI.
PMI reviews and approves (typically 2–3 days after receiving everything).
